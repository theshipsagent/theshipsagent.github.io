Key Architecture Patterns

In-Process DuckDB

No external database server. DuckDB runs in-process, regenerated from scripts. Portable, auditable, no locking concerns for batch ETL workloads.

Multi-Phase Pipelines

Ocean manifests: 10 phases. CBP: 14 steps. Each phase atomic and resumable with intermediate CSV outputs for debugging.

Star Schema (Fact/Dim)

Vessel/port/agent names normalized in dimension tables, referenced by foreign keys in fact tables. Clean, efficient, consistent.

Pluggable Commodities

Core platform is commodity-agnostic. Each commodity module plugs in with its own config, data, and analysis — new commodity added in days.

Golden Record Registry

Site master reconciles EPA FRS (4M+), commodity facilities, and rail SCRS (39,936) into unified facility IDs with source crosswalks.

Regional Drilldowns

Full dataset subsetted for regions (Lower Mississippi, Gulf Coast). Adds domain-specific logic on top of general pipelines. Reusable pattern.
